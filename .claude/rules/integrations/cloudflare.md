# Cloudflare DNS

Cloudflare hosts authoritative DNS for **fnord.net** and **oneill.net** (free
tier). All records are managed by OpenTofu except DDNS and Kubernetes ingress
records (see below).

## OpenTofu Management

Provider: `cloudflare/cloudflare` v5 (`opentofu/main.tf`). Uses **account-level
resources** — the v5 provider replaced user-level resources with account-level
equivalents (`cloudflare_account_token`, etc.).

API token stored in 1Password as `cloudflare-opentofu` (`opentofu/secrets.tf`).

Zone and record definitions:

- `opentofu/modules/dns/fnord.tf` — fnord.net zone and records
- `opentofu/modules/dns/oneill.tf` — oneill.net zone, records, redirect rules
- `opentofu/locals.tf` — infrastructure host definitions (shared with UniFi
  DHCP)

Scoped API tokens for services are minted by OpenTofu and stored in 1Password:

- `cloudflare-ddns` — DNS Write for both zones (`opentofu/cloudflare-ddns.tf`)
- `cloudflare-dns-write` — DNS Write for oneill.net
  (`opentofu/cloudflare-dns-write.tf`), used by external-dns and cert-manager
- `cloudflare-pages-deploy` — Pages Write (`opentofu/cloudflare-pages.tf`)

## Kubernetes Ingress DNS (k.oneill.net)

Kubernetes services are exposed under `k.oneill.net` (a subdomain of
oneill.net). Records are managed by **external-dns**
(`kubernetes/external-dns/`) using the Cloudflare provider.

External-dns watches Kubernetes Ingress resources and automatically creates A
records in the Cloudflare oneill.net zone. It uses the `cloudflare-dns-write`
API token pulled from 1Password via External Secrets.

## Cert-Manager DNS-01 Solver

Cert-manager (`kubernetes/cert-manager/`) uses Cloudflare DNS-01 challenges for
Let's Encrypt certificate issuance. The ClusterIssuer
(`kubernetes/cert-manager/letsencrypt.yaml`) references the
`cloudflare-dns-write` API token for DNS record manipulation.

## DDNS

`favonia/cloudflare-ddns` runs in Kubernetes (`kubernetes/cloudflare-ddns/`).
Updates `luser.fnord.net` and `luser.oneill.net` with the current public IP.

The deployment uses the `cloudflare-ddns` API token (DNS Write, both zones)
pulled from 1Password via External Secrets.

## ESPHome Device DNS

ESPHome devices get Cloudflare A records on the oneill.net zone. These are
generated by `scripts/gen-esphome-hosts` from ESPHome YAML configs and managed
in `opentofu/esphome-hosts.tf`. See `.claude/rules/esphome/static-ips.md` for
the full workflow.
